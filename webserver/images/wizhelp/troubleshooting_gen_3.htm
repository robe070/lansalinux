Wizard does not change the presentation layout of a function
You have created a process layout or modified the default layout, but the functions are not using the new layout or components.

Solution

If you have older functions (created before installing LANSA Release 8.0 EPC538), these functions do not include the commands to include the Wizard layouts. Review the HTML page for your function and search for a <RDML LAYOUT> tag. If this tag is not present, recompile your function so that the HTML page is regenerated. The proper tags will be included.

If you have created your own skeletons in file DC@W22 and you wish to use the e-Business Framework Wizard for these functions, you may need to modify your skeletons according to the default shipped skeleton (WEBSKEL in file DC@F28 in the LANSA data library).

If you are experiencing this problem with a newly created function, use the Editor and check that the correct LAYOUT pages exist for the process in question. If you do not see the <process name>_LAYOUT or DEFAULT_LAYOUT page, execute the Wizard again.
